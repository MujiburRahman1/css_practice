CSS Animations Complete Guide

Note: CSS animations allow you to animate transitions from one CSS style configuration to another.

1. Basic Animation Syntax

/* Define the animation */ @keyframes example { from { background-color: red; } to { background-color: yellow; } } /* Apply the animation */ .element { animation-name: example; animation-duration: 4s; }

2. Animation Properties

animation-name

Specifies the name of the @keyframes animation:

animation-name: slideIn;

animation-duration

How long the animation should take to complete:

animation-duration: 2s;

animation-timing-function

Speed curve of the animation:

animation-timing-function: ease | ease-in | ease-out | ease-in-out | linear | cubic-bezier(n,n,n,n);

animation-delay

Delay before the animation starts:

animation-delay: 1s;

animation-iteration-count

How many times the animation should play:

animation-iteration-count: 3 | infinite;

animation-direction

Whether the animation should play in reverse or alternate cycles:

animation-direction: normal | reverse | alternate | alternate-reverse;

animation-fill-mode

Specifies what values are applied before/after the animation:

animation-fill-mode: none | forwards | backwards | both;

animation (shorthand)

Shorthand for all animation properties:

animation: name duration timing-function delay iteration-count direction fill-mode;

3. Animation Examples

Bounce Animation

@keyframes bounce { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-30px); } } .bounce { animation: bounce 2s infinite; }
Bounce

Rotation Animation

@keyframes rotate { from { transform: rotate(0deg); } to { transform: rotate(360deg); } } .rotate { animation: rotate 3s linear infinite; }
Rotate

Color Change Animation

@keyframes colorChange { 0% { background-color: #3498db; } 50% { background-color: #e74c3c; } 100% { background-color: #2ecc71; } } .color-change { animation: colorChange 4s infinite alternate; }
Colors

Slide-In Animation

@keyframes slideIn { from { transform: translateX(-200px); opacity: 0; } to { transform: translateX(0); opacity: 1; } } .slide-in { animation: slideIn 1.5s forwards; }
Slide In

Pulse Animation

@keyframes pulse { 0% { transform: scale(1); } 50% { transform: scale(1.1); } 100% { transform: scale(1); } } .pulse { animation: pulse 2s infinite; }
Pulse

Mixed Animation

@keyframes mixed { 0% { transform: translateY(0) rotate(0deg); background-color: #3498db; border-radius: 0; } 50% { transform: translateY(-50px) rotate(180deg); background-color: #e74c3c; border-radius: 50%; } 100% { transform: translateY(0) rotate(360deg); background-color: #2ecc71; border-radius: 0; } } .mixed-animation { animation: mixed 4s infinite alternate; }
Mixed

4. Animation Events with JavaScript

// Listen for animation events const element = document.querySelector('.animated-element'); element.addEventListener('animationstart', () => { console.log('Animation started'); }); element.addEventListener('animationend', () => { console.log('Animation ended'); }); element.addEventListener('animationiteration', () => { console.log('Animation iteration'); });

5. Performance Considerations

  • Use transform and opacity for best performance (they don't trigger layout/paint)
  • Avoid animating properties like width, height, margin, etc.
  • Use will-change to hint browsers about upcoming animations
  • Limit the number of simultaneous animations
  • Test animations on mobile devices
